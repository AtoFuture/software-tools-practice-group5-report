## 三、实验过程

### 步骤一：准备安装软件与镜像

1.  下载并安装 **VirtualBox**，安装过程保持默认设置，安装完成后重启电脑。
2.  从阿里云镜像站下载 Ubuntu 22.04 镜像文件：
    - 镜像地址：`https://mirrors.aliyun.com/ubuntu-releases/22.04/ubuntu-22.04.5-desktop-amd64.iso`
    - **注意**：将 `.iso` 文件存放在纯英文路径的文件夹中，避免因路径包含中文导致安装报错。

### 步骤二：新建 Linux 虚拟机

1.  打开 VirtualBox，点击「创建新的虚拟机」。
2.  选择「典型（推荐）」安装模式，浏览并选中下载好的 Ubuntu `.iso` 镜像文件。
3.  设置虚拟机名称（使用英文），存放路径选择 D 盘（非系统盘）。
4.  指定磁盘容量：设置为 **20GB**，并勾选「将虚拟磁盘存储为单个文件」。
5.  点击「自定义硬件」进行配置：
    - **内存**：2GB
    - **处理器**：1 核
    - **网络适配器**：保持默认 **NAT 模式**（满足实验自动联网需求）
6.  点击「完成」生成虚拟机。

### 步骤三：安装 Ubuntu 系统

1.  开启虚拟机，进入 Ubuntu 安装引导界面，语言选择「中文（简体）」。
2.  安装类型选择「清除整个磁盘并安装 Ubuntu」（注：此操作仅针对虚拟磁盘，不影响宿主机 Windows 系统）。
3.  时区选择「上海」。
4.  创建用户账户：输入用户名（英文）并设置登录密码，牢记该密码。
5.  等待安装进度条走完，提示重启后，点击「现在重启」进入系统。

## 六、出现问题与解决

1.  **问题：安装过程中提示路径错误**
    - **原因**：ISO 镜像文件或虚拟机存放路径中包含了中文字符。
    - **解决**：删除已创建的虚拟机，将 ISO 文件移动至纯英文路径（如 `D:\VM_Images`），重新创建虚拟机。
2.  **问题：安装完成后无法上网**
    - **原因**：新建虚拟机时网络适配器误选为「桥接模式」或「仅主机模式」。
    - **解决**：关闭虚拟机，在 VMware 设置中将网络适配器改回 **NAT 模式**，重启虚拟机后恢复正常。
3.  **问题：Ping 命令无法停止**
    - **原因**：Linux 下 `ping` 命令默认会一直执行。
    - **解决**：按下快捷键 `Ctrl + C` 强制终止命令。

1. 熟练掌握 Linux 终端（Terminal）的开启与基础交互。
2. 掌握文件与目录管理的常用命令（`cd`, `ls`, `mkdir`, `touch`, `cp`, `mv`, `rm`）。
3. 学会使用 `apt` 工具进行软件包的更新、安装与卸载。
4. 了解查看系统基础信息（主机名、时间、磁盘、内存）的命令。

- **虚拟化软件**：VMware Workstation / VirtualBox
- **操作系统**：Ubuntu 22.04.5 LTS (Desktop amd64)
- **用户权限**：普通用户（配合 `sudo` 提权）

# Linux 实验五：用户与权限管理实操报告

## 1. 实验目标
掌握 Linux 系统中用户与权限管理的核心操作，具体包括：
- 创建用户组与新建用户，并调整用户归属关系。
- 使用 `chmod` 和 `chown` 命令配置与修改文件权限。
- 查验权限配置效果，理解权限控制的实际作用。

## 2. 前置注意事项
- **密码输入提示**：当命令前带有 `sudo` 且出现密码提示时，输入 `lab4` 的登录密码即可。输入阶段屏幕不会显示光标，正常盲输并回车即可。
- **历史输出查看**：在 TTY 黑窗口中，可使用快捷键 `Shift+PageUp` 和 `Shift+PageDown` 翻页回看历史输出。

## 3. 分步实操指令

### 步骤 1：新建专属用户组
创建测试用户组 `testgroup` 并查验是否创建成功：
```bash
sudo groupadd testgroup
grep testgroup /etc/group
```

### 步骤 2：创建实验用户并归属用户组
创建 3 位实验用户，其中 `user1` 和 `user2` 归属刚建好的组，`user3` 采用默认自建用户组（用于后续对照测试）：
```bash
sudo useradd -m -g testgroup u1
sudo useradd -m -g testgroup u2
sudo useradd -m u3
```
> **说明**：参数 `-m` 会自动生成每位用户对应的 home 家目录。
>![配置](../images/创建用户并归属用户组.png)
### 步骤 3：为新用户设置登录密码
逐条执行以下命令，跟随提示自定义密码（简单测试场景可统一设置为 `123456`）：
```bash
sudo passwd u1
sudo passwd u2
sudo passwd u3
```

### 步骤 4：查看系统保存的用户信息
查验用户与组的创建信息：
```bash
cat /etc/passwd | grep user
cat /etc/group | grep testgroup
```

### 步骤 5：新建测试文件与文件夹
在当前 `lab4` 目录下新建工作目录及测试文件，并查看初始权限：
```bash
mkdir ~/exp5_dir
cd ~/exp5_dir
touch note.txt info.log
ls -l
```
>![配置](../images/新建测试文件与文件夹.png)
> **说明**：执行 `ls -l` 可查看文件的初始权限、所有者及所属组信息。

### 步骤 6：使用 chown 修改文件归属权
将 `demo.txt` 的所有者改为 `u1`，所属组修改为 `testgroup`：
```bash
sudo chown u1:testgroup demo.txt
ls -l demo.txt
```

### 步骤 7：使用 chmod 设置文件权限
**（1）数字权限模式**
设置权限为 `740`（所有者可读、写、执行；所属组只读；其余用户无权限）：
```bash
chmod 740 demo.txt
ls -l demo.txt
```
>![配置](../images/修改文件归属权.png)
**（2）符号权限模式**
给所属组增加写入权限：
```bash
chmod g+w demo.txt
ls -l demo.txt
```

# 实验6：Linux环境配置实验报告

## 一、 实验目的
1. 掌握 Linux 系统中用户级配置文件（如 `~/.bashrc`）的修改与生效方法。
2. 熟悉全局环境变量配置文件（如 `/etc/profile`）的管理，理解用户级配置与全局配置的优先级差异。
3. 学会使用 `alias` 设置命令别名，提升终端操作效率。
4. 了解常见系统配置文件（如 `/etc/passwd`、`/etc/group`）的作用及查看方法。
5. 掌握使用 `hostnamectl` 命令修改系统主机名。

## 二、 实验环境
- **操作系统**：Ubuntu 22.04.5 LTS
- **操作终端**：TTY 
- **当前用户**：lab4（具备 sudo 权限）

## 三、 实验步骤与结果

### 模块1：用户级 Shell 环境变量配置
本模块通过修改当前用户的 `~/.bashrc` 文件，配置专属环境变量、个性化提示符及快捷别名。

1. **编辑配置文件**
使用 `nano` 编辑器打开配置文件：
```bash
nano ~/.bashrc
```

2. **追加自定义配置**
在文件末尾添加以下内容：
```bash
# 自定义专属环境变量
export MY_EXP6_PATH=/home/lab4/exp6_work
# 设置命令提示符个性化样式
export PS1="[\u-Exp6@\h:\W]\$ "
# 配置快捷别名
alias ll='ls -alh'
alias cdd='cd /tmp/exp5_dir'
```

>![配置](../images/自定义配置2.png)
### 6. 实验收尾
3. **加载配置并验证**
保存退出后，执行 `source` 命令使其立即生效：
```bash
source ~/.bashrc
```
验证配置效果：
```bash

ll                  # 检验别名是否生效

```
> *说明：展示修改后的 .bashrc 内容，以及执行 source 后 echo 变量、使用 ll 别名的终端输出。*

### 模块2：全局环境变量配置
本模块通过修改 `/etc/profile` 文件，设置对所有用户生效的全局变量。

1. **编辑全局配置文件**
```bash
sudo nano /etc/profile
```

2. **添加全局变量**
在文件末尾追加：
```bash
export GLOBAL_EXP6="public_variable"
```

3. **重载并验证**
```bash
source /etc/profile
echo $GLOBAL_EXP6
```
---

### 模块3：常用系统配置文件体验
查看系统核心配置文件，了解用户与组的底层存储信息。

1. **查看用户账号配置**
```bash
cat /etc/passwd | grep u1
```

2. **查看用户组配置**
```bash
cat /etc/group | grep group_exp5
```

3. **修改主机名**
```bash
sudo hostnamectl set-hostname exp6-linux
```
---

### 模块4：环境变量生效优先级测试
验证用户级配置（`~/.bashrc`）优先级高于全局配置（`/etc/profile`）的规则。

1. **设置同名变量**
分别在 `/etc/profile` 和 `~/.bashrc` 中写入同名变量但取值不同：
- `/etc/profile` 中：`export TEST_VAR="global_value"`
- `~/.bashrc` 中：`export TEST_VAR="user_value"`

2. **加载并观察**
```bash
source /etc/profile
source ~/.bashrc
echo $TEST_VAR
```
*结果：终端输出 `user_value`，证明用户级配置覆盖了全局配置。*


## 四、 实验总结
通过本次实验，我深入理解了 Linux 环境变量的加载机制：
1. **配置文件层级**：`/etc/profile` 作用于全局，`~/.bashrc` 作用于当前用户。当两者存在冲突时，后加载的用户级配置优先级更高。
2. **别名与提示符**：通过 `alias` 和 `PS1` 变量可以极大地定制化终端体验，提高日常运维效率。
3. **系统底层认知**：通过查看 `/etc/passwd` 等文件，直观地看到了 Linux 用户管理的底层数据结构，加深了对系统账户体系的理解。

在实验过程中，修改 `/etc` 目录下的文件必须使用 `sudo` 提权，且修改配置后务必使用 `source` 命令重载，否则配置不会立即生效。

3. 配置Nginx站点：

```bash
sudo nano /etc/nginx/sites-available/webexp8
```
>![配置](../images/安装nginx.png)
填入以下配置内容：

```nginx
server {
    listen 80;
    server_name 虚拟机静态IP;
    root /var/www/website;
    index index.html;
    location / {
        try_files $uri $uri/ =404;
    }
}
```

4. 启用站点配置并重载Nginx：

```bash
sudo ln -s /etc/nginx/sites-available/webexp8 /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

5. 放行虚拟机防火墙的80网页端口：

```bash
sudo ufw allow 80/tcp
```

## 四、常见问题与解决
1. **浏览器打不开网页**：首先核验虚拟机防火墙是否放行80端口，接着检查Nginx配置语法是否合法、网页目录权限是否适配 `www-data` 用户。
2. **Git拉取文件提示权限报错**：对站点目录重新执行 `chown` 权限指令，切勿直接使用宽泛的777权限。
3. **页面样式失效**：检查css、图片资源的相对路径书写是否正确。
4. **迭代更新线上站点没有变化**：确认虚拟机侧已经执行 `git pull` 拉取最新代码，必要时清理浏览器缓存重试。
